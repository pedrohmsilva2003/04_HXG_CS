Contexto
Por que precisamos disso?
Sem um sistema formal, horas técnicas ficam invisíveis.
📋
Sem rastreabilidade
Não há como saber quanto tempo foi gasto em cada OS. Relatórios são feitos de memória ou retroativamente.
👥
Gestor sem visibilidade
Impossível saber em tempo real quem está trabalhando em quê, ou se algum técnico ficou ocioso.
📉
Utilização desconhecida
Meta de 44h/semana existe, mas não há como medir se está sendo atingida ou onde as horas estão sendo gastas.
📂
Dados para análise externa
Planilhas manuais inconsistentes. Não há base de dados estruturada para cruzar com OS do ERP.
Visão geral
O módulo em 3 pilares
Cada pilar resolve uma parte do problema.
⏱
Registrar
Técnico inicia, pausa, retoma e finaliza atividades pelo navegador. Timer roda em tempo real.
📊
Monitorar
Gestor vê em tempo real quem está ativo, em qual OS, há quanto tempo — com auto-refresh a cada minuto.
📥
Analisar
Relatórios com filtros avançados e exportação Excel com 4 abas para análise externa.
Tudo armazenado no Supabase — dados persistem mesmo que o navegador feche. Cada técnico opera de forma totalmente independente.
Fluxo de trabalho
Como o técnico registra

Setup inicial
Como ativar o módulo
Apenas 3 passos — o módulo já está no ar.
1

Execute o SQL no Supabase

Abra o SQL Editor e execute o arquivo supabase_horas_schema.sql. Cria as 5 tabelas.

supabase.com/dashboard/project/nkohoaygnnpvmjattzit/sql/new
2

Acesse o app CS

O módulo já substituiu o placeholder "Em desenvolvimento". Faça login ou venha pelo SSO do Portal.

3

Inicie o primeiro apontamento

Informe a OS, selecione o serviço e clique em Iniciar Apontamento.

✓

Opcional: importe a base de OS

Insira os dados da planilha na tabela base_administrativa_cs para habilitar a busca automática.
